# Условие

Разработать FastCGI-сервер на языке Java, определяющий попадание точки на координатной плоскости в заданную область, и создать HTML-страницу, которая формирует данные для отправки их на обработку этому серверу.

Параметр `R` и координаты точки должны передаваться серверу посредством HTTP-запроса. Сервер должен выполнять валидацию данных и возвращать HTML-страницу с таблицей, содержащей:

- полученные параметры;
- результат вычислений — факт попадания или непопадания точки в область.

Допускается вместо HTML-страницы возвращать JSON-строку.

Предыдущие результаты должны сохраняться между запросами и отображаться в таблице.

Кроме того, ответ должен содержать:

- данные о текущем времени;
- время работы скрипта.

## Требования к выполнению лабораторной работы

- Требуется поднять Apache `httpd` веб-сервер от лица своего пользователя на гелиосе.
- Шаблон файла конфигурации доступен для скачивания наверху страницы.
- Веб-сервер должен обслуживать статический контент (`HTML`, `CSS`, `JavaScript`) и перенаправлять запросы за динамическим контентом к FastCGI-серверу.
- FastCGI-сервер требуется реализовать на языке Java и поднять также на гелиосе.
- Полезная библиотека в виде `jar`-архива доступна для скачивания наверху страницы.
- Путём обращений из JavaScript к FastCGI-серверу требуется показать понимание принципа `AJAX`.

## Требования к HTML-странице

Разработанная HTML-страница должна удовлетворять следующим требованиям:

1. Для расположения текстовых и графических элементов необходимо использовать **табличную вёрстку**.
2. Данные формы должны передаваться на обработку посредством **GET-запроса**.
3. Таблицы стилей должны располагаться в **отдельных файлах**.
4. При работе с CSS должно быть продемонстрировано использование:
   - селекторов классов;
   - селекторов потомков;
   - селекторов элементов;
   - селекторов псевдоклассов;
   - наследования;
   - каскадирования.
5. HTML-страница должна иметь **шапку**, содержащую:
   - ФИО студента;
   - номер группы;
   - номер варианта.
6. При оформлении шапки необходимо явным образом задать в CSS:
   - шрифт `serif`;
   - цвет шрифта;
   - размер шрифта.
7. Отступы элементов ввода должны задаваться **в пикселях**.
8. Страница должна содержать сценарий на языке JavaScript, осуществляющий **валидацию** значений, вводимых пользователем в поля формы.
9. Любые некорректные значения должны блокироваться, например:
   - буквы в координатах точки;
   - отрицательный радиус.

## Что требуется реализовать

Нужно разработать:

- FastCGI-сервер на Java;
- HTML-страницу для ввода данных;
- CSS-файлы для оформления страницы;
- JavaScript-код для клиентской валидации и AJAX-запросов;
- механизм сохранения предыдущих результатов между запросами.

## Результат работы

Система должна:

- принимать координаты точки и параметр `R`;
- проверять корректность входных данных;
- определять, попадает ли точка в заданную область;
- возвращать результат вместе с текущим временем и временем выполнения;
- сохранять и отображать предыдущие результаты в таблице.
